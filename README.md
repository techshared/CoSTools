## 为什么要做 CoSTools

**打卡制正在失效。** 调研显示：
- 打卡制员工日均有效工作时长仅 **4.2h**，离职率 **35%/年**
- 不打卡团队日均有效工作时长 **7.8h**，离职率 **4%/年**
- 不打卡模式人效高 **60%**，招聘成本低 **40%**

但市面上的管理软件全是"监控-管控"思维——考勤、定位、屏幕截图、鼠标追踪……这些与不打卡组织的**信任文化**背道而驰。

**CoSTools 反其道而行：不监控、不打卡、不追踪，只看成果。**

---

## 四大模块

### ¥ 收益分成计算引擎
```
项目利润 → 自动分成 → 全员透明
```
- 项目利润 **30%** 自动分配，权重由团队共同决定
- 分成明细**全员可见**，每一笔都可追溯审计
- 支持审批流，防止错分漏分

### ⚡ 异步决策记录器
```
飞书发条消息 → 自动记录 → 随时搜索
```
- 在飞书 Bot 输入 `记录决策：标题 - 理由`，一句话完成记录
- 支持**全文搜索**，历史决策一秒找到
- 决策生命周期管理（进行中 → 已完成 → 已废弃）

### ◈ 自驱力仪表盘
```
不追踪个人，只看团队健康度
```
- **7 大健康指标**：OKR 进展、任务完成率、协作密度、文档活跃度等
- **隐私优先**：个人数据仅自己可见，绝不追踪时长/鼠标/屏幕
- 支持接入 GitHub、飞书审批等外部数据源，自动生成趋势分析

### ◎ 成员贡献图谱
```
代码·文档·评审·设计——多维度贡献可视化
```
- 基于 **PageRank 算法**，从代码提交、文档撰写、代码评审等维度综合评分
- **协作网络图**直观展示谁在和谁协作、贡献流向
- 与收益分成引擎联动，贡献分直接影响分成权重

## 和传统管理软件有什么区别？

| | 传统 SaaS | CoSTools |
|---|-----------|----------|
| 考勤 | 打卡/定位/迟到扣钱 | **不做** |
| 监控 | 屏幕截图/鼠标追踪/活跃度 | **不做** |
| 考核 | 上级打分/KPI 扣分制 | 全员可见的分成+图谱 |
| 决策 | 线下开会/口头传达 | 飞书 Bot 记录，永久可查 |
| 协作 | 封闭审批流 | 开放权重投票+图谱 |
| 隐私 | 个人数据归公司 | 个人数据仅自己可见 |
| 部署 | SaaS 订阅 / 数据上云 | 自部署，数据自主可控 |

## 谁适合用 CoSTools？

- **小微创业团队**（5-50 人）— 不上打卡，但要上分成和决策记录
- **远程/混合办公团队** — 不用管人在哪，只看交付成果
- **项目制团队** — 项目利润按贡献自动分成
- **开源社区/DAO** — 贡献可视化 + 透明分配

## 快速安装

> 需要：Docker + 一个域名 + 2 分钟

```bash
# 1. Clone
git clone https://github.com/techshared/CoSTools.git
cd costools

# 2. 配置环境
cp .env.example .env
# 填入飞书应用凭证（在 open.feishu.cn 免费创建）

# 3. 一键启动
docker compose up -d

# 4. 播种演示数据
python3 scripts/seed-data.py
```

### 配置 Cloudflare Tunnel（公网访问）

```bash
cloudflared tunnel login
cloudflared tunnel create costools
cloudflared tunnel route dns costools 你的域名
cloudflared tunnel run costools
```

### 在飞书中使用

1. 在[飞书开放平台](https://open.feishu.cn)创建自建应用
2. 配置事件请求网址：`https://你的域名/webhook/event`
3. 配置网页应用主页：`https://你的域名`
4. 开启机器人能力并发布

之后在飞书工作台就能看到 CoSTools，Bot 也上线了。
